# Contributing to DXRP

Thank you for your interest in contributing to DXRP.

## Read this first

DXRP is **source-available**, not open source.

Although the code, models, assets, packages, or related game materials may be publicly visible or technically accessible through s&box, GitHub, downloads, or other third-party platforms, all rights are reserved except for the limited permissions in [LICENSE.md](LICENSE.md).

Please read [LICENSE.md](LICENSE.md) before contributing.

## What contributions are welcome

We welcome:
- bug fixes
- documentation improvements
- focused usability enhancements
- feature work linked to an approved issue
- performance improvements
- security fixes submitted through the proper channel
- bounty work where Dxura has explicitly posted or assigned a bounty

Please avoid submitting large unsolicited features without an approved issue first. For feature proposals, open or comment on an issue so scope, ownership, and acceptance criteria can be agreed before implementation.

## Bounties

Dxura may occasionally attach bounties to specific issues. A bounty is only valid when it is explicitly posted, assigned, or confirmed by Dxura through an approved project channel.

Unless Dxura agrees otherwise in writing, submitting work for a bounty does not guarantee acceptance, merge, release, attribution, or payment. Bounty eligibility, review, acceptance, and payout terms are determined by Dxura for the specific issue.

## Contribution rules

By contributing, you agree that:
- you have the legal right to submit the contribution;
- the contribution is your original work, or you otherwise have sufficient rights to submit it;
- your contribution does not knowingly infringe any third-party rights;
- Dxura may use, modify, relicense, distribute, commercialize, and otherwise exploit your contribution under the project's applicable terms.

By submitting a pull request, patch, or other contribution to DXRP through an accepted project channel, you agree to the Contributor License Agreement in [CLA.md](CLA.md).

## No redistribution

Contributing does **not** grant permission to:
- repost the repository contents elsewhere;
- publish forks except as permitted for pull request submission;
- publish mirrors or archives;
- distribute modified versions;
- release binaries, game packages, asset packs, or derived builds;
- run a hosted public service based on the project except for dedicated servers operated through Dxura-provided workflows where available;
- create or operate a competing commercial or public clone.

If you want any of those rights, contact Dxura for written permission.

If public pull requests are enabled, you may create a fork only as needed to prepare and submit a contribution back to Dxura. That fork may not be used to publish, distribute, host, operate, or promote a separate version of DXRP.

Dedicated server hosting is allowed only when using Dxura-provided server workflows, images, packages, or documentation, and only within the limits stated by those workflows.

## Submission process

Submit changes only through channels approved by Dxura.

If pull requests are enabled, submitting a pull request means you agree to the contribution terms, [CLA.md](CLA.md), and [LICENSE.md](LICENSE.md).

If an issue template, patch process, or review workflow is provided, follow that process.

## Review policy

All contributions are reviewed at the sole discretion of Dxura.
Submission does not guarantee review, acceptance, merge, release, attribution, compensation, or continued availability.

## Security reporting

Do not disclose security vulnerabilities publicly before coordinated review.
Use the project's designated private security reporting channel if one is provided.

## Branding and clones

Please do not publish copies, mirrors, or clone projects derived from DXRP.
Unauthorized redistributions may be subject to takedown requests or other enforcement.
